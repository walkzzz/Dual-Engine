### 2. 性能优化（引擎类项目核心痛点）
#### （1）对象池优化（减少频繁内存分配）
引擎中渲染/物理对象频繁创建销毁会导致性能损耗，实现通用对象池：
```cpp
// C++示例：线程安全的对象池模板
template <typename T>
class ObjectPool {
private:
    std::vector<std::unique_ptr<T>> pool;  // 存储所有对象
    std::queue<T*> available;             // 可用对象队列
    std::mutex mtx;                       // 线程安全锁

public:
    // 获取对象
    T* acquire() {
        std::lock_guard<std::mutex> lock(mtx);
        if (available.empty()) {
            pool.emplace_back(std::make_unique<T>());
            available.push(pool.back().get());
        }
        T* obj = available.front();
        available.pop();
        return obj;
    }

    // 回收对象
    void release(T* obj) {
        std::lock_guard<std::mutex> lock(mtx);
        available.push(obj);
        // 可选：重置对象状态
        obj->reset();
    }

    // 清理池
    ~ObjectPool() {
        std::lock_guard<std::mutex> lock(mtx);
        pool.clear();
        while (!available.empty()) available.pop();
    }
};

// 使用示例：GameObject池
ObjectPool<GameObject> gameObjPool;
GameObject* obj = gameObjPool.acquire();
// ... 使用对象 ...
gameObjPool.release(obj);
```

#### （2）空间划分优化（物理碰撞检测）
引擎碰撞检测默认全量遍历效率低，用**四叉树（2D）/八叉树（3D）** 减少检测范围：
```cpp
// 简化版2D四叉树（碰撞检测优化）
class QuadTree {
private:
    Rect bounds;                  // 四叉树边界
    std::vector<Collider*> objs;  // 当前节点对象
    std::array<std::unique_ptr<QuadTree>, 4> children; // 子节点
    bool divided = false;
    const int MAX_OBJS = 4;       // 节点最大对象数

    // 分割为4个子节点
    void subdivide() {
        float x = bounds.x, y = bounds.y;
        float w = bounds.w / 2, h = bounds.h / 2;
        children[0] = std::make_unique<QuadTree>(Rect{x+w, y, w, h});  // 右上
        children[1] = std::make_unique<QuadTree>(Rect{x, y, w, h});    // 左上
        children[2] = std::make_unique<QuadTree>(Rect{x, y+h, w, h});  // 左下
        children[3] = std::make_unique<QuadTree>(Rect{x+w, y+h, w, h});// 右下
        divided = true;
    }

public:
    explicit QuadTree(Rect b) : bounds(b) {}

    // 插入碰撞体
    void insert(Collider* collider) {
        if (!bounds.contains(collider->getRect())) return;
        if (objs.size() < MAX_OBJS && !divided) {
            objs.push_back(collider);
            return;
        }
        if (!divided) subdivide();
        for (auto& child : children) child->insert(collider);
    }

    // 查询指定范围内的碰撞体
    std::vector<Collider*> query(Rect range) {
        std::vector<Collider*> res;
        if (!bounds.intersects(range)) return res;
        // 检查当前节点对象
        for (auto* obj : objs) {
            if (range.intersects(obj->getRect())) res.push_back(obj);
        }
        // 递归检查子节点
        if (divided) {
            for (auto& child : children) {
                auto childRes = child->query(range);
                res.insert(res.end(), childRes.begin(), childRes.end());
            }
        }
        return res;
    }
};
```

#### （3）多线程异步优化（耗时操作解耦）
将资源加载、物理计算等耗时操作放到后台线程，避免阻塞主线程：
```cpp
// C++线程池（复用线程，避免频繁创建）
class ThreadPool {
private:
    std::vector<std::thread> workers;
    std::queue<std::function<void()>> tasks;
    std::mutex mtx;
    std::condition_variable cv;
    bool stop = false;

public:
    explicit ThreadPool(size_t threadNum = std::thread::hardware_concurrency()) {
        for (size_t i = 0; i < threadNum; ++i) {
            workers.emplace_back([this] {
                while (true) {
                    std::function<void()> task;
                    {
                        std::unique_lock<std::mutex> lock(mtx);
                        cv.wait(lock, [this] { return stop || !tasks.empty(); });
                        if (stop && tasks.empty()) return;
                        task = std::move(tasks.front());
                        tasks.pop();
                    }
                    task(); // 执行任务
                }
            });
        }
    }

    // 提交任务
    template<class F>
    void enqueue(F&& f) {
        {
            std::lock_guard<std::mutex> lock(mtx);
            tasks.emplace(std::forward<F>(f));
        }
        cv.notify_one();
    }

    ~ThreadPool() {
        {
            std::lock_guard<std::mutex> lock(mtx);
            stop = true;
        }
        cv.notify_all();
        for (auto& worker : workers) worker.join();
    }
};

// 使用示例：异步加载纹理
ThreadPool pool;
pool.enqueue([]() {
    Texture* tex = ResourceManager::loadTexture("assets/terrain.png");
    // 加载完成后通知主线程
    MainThreadDispatcher::post([tex]() {
        Renderer::registerTexture("terrain", tex);
    });
});
```

### 3. 工程化与CI/CD优化
#### （1）GitHub Actions 自动化构建/测试
创建 `.github/workflows/ci.yml`，实现代码推送/PR时自动构建、测试、代码检查：
```yaml
name: CI/CD

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  build-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      # 配置编译环境（C++示例）
      - name: Install Dependencies
        run: sudo apt-get install -y cmake clang libgtest-dev
      - name: Configure CMake
        run: cmake -B build -DCMAKE_CXX_COMPILER=clang++ -DCMAKE_BUILD_TYPE=Release
      - name: Build
        run: cmake --build build --parallel 4
      - name: Run Tests
        run: cd build && ctest --output-on-failure
      - name: Code Style Check
        run: clang-format --dry-run --Werror src/**/*.cpp src/**/*.h
```

### 5. 安全与依赖优化
#### （1）安全编译选项（C++）
在 `CMakeLists.txt` 中添加安全编译标志，减少内存越界、栈溢出等风险：
```cmake
if (CMAKE_CXX_COMPILER_ID MATCHES "GNU|Clang")
    target_compile_options(dual-engine PRIVATE
        -Wall -Wextra -Wpedantic -Werror=return-type
        -fstack-protector-strong -D_FORTIFY_SOURCE=2
        -O2 -fPIE
    )
    target_link_options(dual-engine PRIVATE -pie -z relro -z now)
elseif (MSVC)
    target_compile_options(dual-engine PRIVATE
        /W4 /WX /GS /sdl /O2
    )
endif()
```
